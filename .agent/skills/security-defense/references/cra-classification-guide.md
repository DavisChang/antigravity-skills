# CRA SKU 分類工作坊指南

> 每個出口 EU 的產品 SKU 都必須完成此分類，才能決定後續的符合性評估路徑

---

## 分類工作坊：每個 SKU 回答 5 題

### Q1. 核心功能 (Core Functionality) 是什麼？

CRA 判定產品類別時，看的是 **core functionality**，不是產品附帶做了多少其他功能。

```
範例：
- 防火牆 → 核心功能：網路流量過濾與存取控制 → 落 Annex III Class II
- 路由器 → 核心功能：網路封包路由 → 落 Annex III Class I
- 智慧門鎖 → 核心功能：門禁控制 → 可能落 Class I（具安全功能的智慧家居產品）
- 健身手環 → 核心功能：健康數據追蹤 → 需判定是否落 Annex III
```

**填寫：**

| SKU 名稱 | 產品型號 | 核心功能描述 |

| | | |

### Q2. 是否落入 Annex III / IV 類別？

#### Annex III — Important Products Class I（典型範例）

- [ ] 身份管理系統和特權訪問管理軟硬體
- [ ] 獨立和嵌入式瀏覽器
- [ ] 密碼管理器
- [ ] 搜索/刪除/隔離惡意軟體的軟體
- [ ] 具 VPN 功能的產品
- [ ] 網路管理系統
- [ ] SIEM 系統
- [ ] Boot managers
- [ ] 公開金鑰基礎設施和數位憑證軟體
- [ ] 物理和虛擬網路介面
- [ ] **作業系統**
- [ ] **路由器、數據機和交換機**（連接互聯網用）
- [ ] 具安全相關功能的微處理器
- [ ] 具安全相關功能的微控制器
- [ ] 具安全相關功能的 ASIC 和 FPGA
- [ ] 智慧家庭通用虛擬助手
- [ ] **具安全功能的智慧家居產品**（門鎖、監視器、嬰兒監控、警報）
- [ ] 具社交互動或位置追蹤功能的互聯網連接玩具
- [ ] 健康監測穿戴產品（非醫療器械）或兒童穿戴產品

#### Annex III — Important Products Class II（典型範例）

- [ ] **支援虛擬化的虛擬機器管理程式和容器執行時系統**
- [ ] **防火牆、入侵偵測和預防系統**
- [ ] 防篡改微處理器
- [ ] 防篡改微控制器

#### Annex IV — Critical Products

（目前清單較短，通常由 Commission 更新）

### Q3. 是否依賴 App / Cloud / Backend 完成核心功能？

**CRA 範圍包含 remote data processing solutions：**

| SKU | 是否需要手機 App | 是否需要雲端服務 | 是否需要帳號系統 | 是否需要 OTA 平台 | 是否有第三方 API 依賴 |
|-----|-----------------|-----------------|-----------------|------------------|---------------------|
| | 是/否 | 是/否 | 是/否 | 是/否 | 是/否 |

**判定規則：**
- 若上述任一項是核心功能不可或缺的一部分 → 一起落入 CRA 邊界
- 若僅是增值功能且產品可獨立運作 → 可不納入
- 若不確定 → 建議先納入

### Q4. Support Period 要設幾年？

| 因素 | 考量 |
|------|------|
| CRA 最低要求 | **至少 5 年** |
| 產品預期使用年限 | 若超過 5 年，support period 需更長 |
| 網通設備特別注意 | CRA FAQ 點名 router/modem/switch 通常需要 **超過 5 年** |
| 晶片/BSP 供應商支援期 | 供應商 EOL 是否早於你的 support period |
| 客戶合約與期待 | B2B 客戶可能期望 7-10 年 |

**填寫：**

| SKU | 預期使用年限 | 設定 Support Period | BSP/晶片供應商支援到 | 是否有缺口 |
|-----|-------------|--------------------|--------------------|-----------|
| | | 年 | | 是/否 |

## 分類後的下一步

| **不確定** | 先按較高類別準備，待標準明確後調整 |
