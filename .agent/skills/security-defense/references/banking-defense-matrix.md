# Banking & Financial Sector — Attack-Defense Matrix

> 8 條攻擊主線對應偵測點、防禦控制、演練方式與 KPI

---

## 1. 帳號接管 (ATO) / 員工釣魚 / AiTM / MFA 疲勞轟炸

### 常見手法
- 郵件釣魚、AiTM 登入中間人
- 憑證填充、密碼重用
- MFA fatigue / push bombing
- 竊取 session / token
- 針對 O365、SSO、VPN、VDI、管理後台

### 影響資產
員工信箱、遠端存取、AD/IAM/SSO、管理後台、核心金融系統入口

### 偵測點
- 不尋常地理位置登入
- 同帳號短時間多次失敗 / 多次 MFA push
- 不同裝置快速切換
- 新裝置登入後立刻查詢高敏感資料
- Token 異常延續或 refresh 異常增長

### 防禦控制
- 對郵件、VPN、SSO、特權帳號全面導入 phishing-resistant MFA
- 停用弱 MFA 作為高風險系統唯一驗證因子
- 條件式存取、裝置信任、風險式驗證
- 特權帳號獨立管理，禁止共用帳號
- 新裝置 / 高風險操作加上 step-up 驗證

### 演練方式
- 模擬釣魚信點擊到登入竊取
- 模擬 MFA 疲勞轟炸
- 模擬已登入帳號被劫持後的橫向移動
- 驗證 SOC 是否能在 15 分鐘內識別並停權

### KPI
- 高風險系統 phishing-resistant MFA 覆蓋率
- 特權帳號無共用比例
- 可疑登入到帳號封鎖中位時間
- MFA 疲勞事件月趨勢

## 2. 邊界設備 / VPN / 反向代理 / 對外暴露面漏洞利用

### 常見手法
- 利用 VPN / WAF / 反向代理 / 檔案傳輸設備漏洞
- 打舊版管理介面、弱 TLS、預設設定
- 從外網進入後再橫向移動

### 影響資產
外網入口、遠端維運、入口閘道、核心區前置層

### 偵測點
- 新增或異常的外網服務暴露
- 邊界設備異常流量/程序/設定變更
- 管理介面被外網存取
- 同一 IP 對多入口探測
- 新出現的高風險 CVE 命中資產

### 防禦控制
- 建立每日更新的外網資產 inventory
- 邊界設備設更短修補 SLA
- 管理平面與業務平面隔離
- 對高風險 edge device 做虛擬補丁與暴露縮減
- 定期驗證不必要 port、管理介面、debug 端點是否對外

### KPI
- 高風險 edge 資產修補中位時間
- 無 owner 的外網資產數量
- 對外管理介面數量
- 高風險 CVE 暴露天數

## 3. DDoS / Hacktivism / 網銀與 App 可用性攻擊

### 常見手法
- Layer 3/4 流量洪泛
- Layer 7 HTTP/HTTPS 請求洪泛
- 對登入、查詢、OTP、帳務 API 做應用層阻斷
- 配合社群宣傳製造信任危機

### 影響資產
網銀入口、行動銀行 API、登入/查詢/轉帳服務、客服入口與公告站

### 偵測點
- 邊界頻寬突升
- 特定 URI / API path 請求異常
- 來自特定 ASN / 地區的大量短連線
- CDN/WAF 被大量 challenge 命中
- 應用層資源耗盡、排隊時間飆升

### 防禦控制
- 與 CDN / WAF / DDoS 清洗服務整合
- 對登入、OTP、查詢與交易 API 做分層 rate limiting
- 建立降級模式：唯讀查詢、延後非關鍵功能、靜態公告切換
- 把關鍵 API 與非關鍵流量做容量隔離
- 預先定義對外公告、客服與監理通報流程

### KPI
- TTD / TTM（偵測與緩解時間）
- 高峰期間登入成功率
- 受攻期間核心交易成功率
- 降級切換時間

## 4. API 越權 / BOLA / 功能層授權錯誤 / 流程濫用

### 常見手法
- 改 object ID 查別人資料
- 一般用戶呼叫管理功能
- 大量打 OTP / 驗證 / 查詢流程
- 舊版 API、debug endpoint、未下線測試端點
- SSRF 打內部服務

### 影響資產
客戶資料 API、交易/轉帳/查詢 API、管理 API、報表下載/匯出、第三方串接 API

### 偵測點
- 同帳號對大量 object ID 嘗試
- 權限失敗後快速切換 path / method
- OTP / SMS / email 驗證短時間高頻觸發
- 大量 403 / 429 / 401 組合出現
- 內網位址型態的 URI 被 API 嘗試抓取

### 防禦控制
- 每支 API 做物件層與功能層授權檢查
- 高價值 API 建立 owner、版本、資料分類與風險分級
- 對登入、OTP、查詢、下載做 per-user / per-IP / per-device 限流
- 關閉舊版 API、swagger/debug/playground
- SSRF 防護：目的地白名單、metadata/內網位址封鎖

### KPI
- 高風險 API 授權測試覆蓋率
- 無 owner API 數量
- 舊版 API 下線率
- 每季 BOLA / BFLA 缺陷數

## 5. 勒索 / 端點入侵 / 橫向移動 / 還原能力

### 常見手法
- 先偷帳號，再進端點與 AD
- 部署惡意工具、停用防護、橫向移動
- 竊資料後加密或純勒索
- 先打備份、虛擬化與管理主機

### 影響資產
AD / IAM、伺服器與端點、備份系統、檔案分享、核心交易支撐系統

### 偵測點
- 大量檔案重新命名 / 加密行為
- 端點防護停用
- 異常權限提升、橫向登入
- 大量壓縮、分卷、外傳
- 備份刪除、快照異常、還原點消失

### 防禦控制
- 備份離線化、不可變、加密化
- 定期還原演練
- 特權分離、AD hardening、伺服器分段
- EDR 覆蓋與 tamper protection
- 對關鍵系統設快速隔離與斷網程序

### KPI
- 關鍵系統還原成功率
- 還原時間達標率
- 不可變備份覆蓋率
- 高風險資產分段覆蓋率

---

## 6. 第三方 / 供應鏈 / 外包維運 / 洩密憑證

### 常見手法
- 供應商帳號被釣魚或憑證重用
- 開發或維運金鑰外流
- 第三方系統成為跳板
- 遠端維運入口被濫用

### 影響資產
第三方整合 API、外包維運入口、CI/CD / Git / Artifact、跳板機 / 遠端桌面

### 偵測點
- 供應商帳號在非維運時段登入
- 服務帳號出現互動式登入
- Git / repo / pipeline 出現 secrets
- 第三方來源 IP 模式異常
- 新增整合後權限快速擴張

### 防禦控制
- 第三方帳號獨立治理與最小權限
- 維運入口設 JIT / JEA 與錄影稽核
- Secrets 掃描、輪替與短生命週期憑證
- 第三方連線強制 MFA 與來源限制
- 契約納入事件通報與補救時限

### KPI
- 第三方高權限帳號數量
- Secrets 輪替中位時間
- 供應商登入錄影覆蓋率
- 第三方事件通報達 SLA 比例

## 7. Swift / 支付訊息 / 特權操作面

### 常見手法
- 針對 Swift 關聯主機、操作站、跳板與特權帳號
- 偽冒操作、未授權變更
- 對支付訊息與操作流程下手
- 借道一般 IT 環境再打到 Swift 環境

### 影響資產
Swift 關聯伺服器、操作與審核終端、支付訊息流程、高權限操作帳號

### 防禦控制
- Swift 專區網段與一般 IT 嚴格隔離
- 強化特權存取、雙人覆核、白名單操作
- 對 Swift 系統做獨立日誌與稽核
- 依 CSCF 控制項做獨立評估與補強
- 把支付異常監控與 SOC 事件串起來

### KPI
- CSCF 控制符合率
- Swift 環境獨立評估完成率
- 特權操作雙人覆核覆蓋率
- Swift 區與一般區違規流量數

---

## 8. 資料外洩 / 大量查詢 / 內部濫用 / 報表匯出

### 常見手法
- 合法帳號做異常大量查詢
- 報表 / 匯出功能被濫用
- 過度授權欄位直接暴露
- 內部人員或受害帳號做批量存取

### 影響資產
客戶資料庫、報表平台、後台查詢系統、Data API / BI 介面

### 偵測點
- 查詢量突然暴增
- 單帳號跨大量客戶 ID 存取
- 短時間大量下載 / 匯出
- 非上班時段大批查詢
- 查詢模式與角色不符

### 防禦控制
- 欄位級遮罩與最小揭露
- 高敏查詢加用途與理由記錄
- 下載 / 匯出加上 step-up 與審批
- UEBA / DLP 針對查詢與匯出行為建模
- 敏感資料 API 做 object/property 層授權

### KPI
- 大量查詢事件 TTD
- 匯出審批覆蓋率
- 欄位遮罩覆蓋率
- 每季異常查詢封鎖率

## 90 天優先行動

### 0-30 天
- 盤點所有高風險入口（郵件、SSO、VPN、管理後台、第三方維運）
- 確認哪些系統還沒上 phishing-resistant MFA
- 建立外網資產總表與 owner
- 列出所有高價值 API 與版本
- 盤點備份是否離線、不可變、可還原

### 31-60 天
- 邊界設備修補 SLA 與每週例檢
- 登入、OTP、查詢、下載 API 上線 rate limiting 與告警
- DDoS 方案做桌演與切換演練
- 第三方維運帳號做最小權限與登入來源限制

### 61-90 天
- 紫隊演練：釣魚 → 帳號接管 → 橫向移動 → API 濫用 / 匯出
- 營運韌性演練：DDoS / 勒索 / 核心服務還原
- 以 NIST CSF 2.0 做出 Current Profile / Target Profile / Gap list
- 把情資、SOC 規則、修補優先順序與董事會報表串起來
